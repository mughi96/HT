Hash Table

The idea behind a Hash Table is that for each element we want to store, we calculate a unique address and we put the value at this index in the array. When we need to find a value, we once again calculate its index and then return the value. In other words, a Hash Table allows us to store and retrieve objects by key.


In this visualisation, there is a Hash Table of length 17. Each index of the Hash Table contains a pointer to a secondary array of length 3. The purpose of the secondary array is to store collisions (when multiple strings have the same hash value).


Hash Function

A hash function generates new values according to a mathematical hashing algorithm, known as a hash value or simply a hash. In this visualisation, the hash function used is a Polynomial Rolling Hash Function adopted from the Rabin-Karp algorithm. It is displayed below the hash table.

Simply put, the Hash Function takes in a string as input and gives an index of the Hash Table as the output. The string is then stored in the corresponding array of that index.

In the Hash Equation used in this Hash Table,

  • a = 7919 (a random large prime number)
  • c = character code
  • k = string length

How To Use This Tool

This tool is meant to demonstrate the basic working principles of a Hash Table. The Hash Table is displayed on the left, with the Hash Function equation displayed below it. The animation displaying the steps taken by the Hash Function will be displayed below the Hash Function equation after a string has been input.


There are 3 main operations featured in this tool:
  1. Insert

  2. Delete

  3. Search/Find

Scroll down along this box for instructions on how to execute the operations.

Insert a String

To insert a string into the Hash Table:


  1. In the 'INSERT' input box, enter a string of maximum 10 characters

  2. Press 'Insert'

  3. The individual steps taken by the hashing algorithm will be displayed under the Hash Function equation below the Hash Table

  4. Once the hashing is done, the string will be appear in an index of the Hash Table corresponding to its hash value

  5. If the string is hashed to an index that already contains a string, it will be stored at the next available slot of that index, up to a maximum of 3 strings

  6. The Insert operation is complete

Some examples of strings that can be used are:

  • "hello"
  • "morning"
  • "TORTOISE"

Some examples of colliding strings are:

  • "new" & "purple" & "jacket"
  • "chicken" & "fish"
  • "beef" & "gold"

The string can include uppercase, lowercase, integers and special characters.

Delete a String

To delete a string from the Hash Table:


  1. In the 'DELETE' input box, enter a string that is already in the Hash Table

  2. Press 'Delete'

  3. The string goes through the same process as the Insert operation, but the hashing is not displayed as it happens in the background

  4. The corresponding string in the Hash Table is then removed

  5. The Delete operation is complete

Find a String

To search for a string in the Hash Table:


  1. In the 'SEARCH' input box, enter a string

  2. Press 'Find'

  3. The string goes through the hash function in the background and if the same string is found in the Hash Table, the corresponding index will be highlighted in red

  4. If no such string exists, nothing will happen

  5. The Search/Find operation is complete
